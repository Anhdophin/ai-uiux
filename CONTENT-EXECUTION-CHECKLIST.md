# Content Execution Checklist

Use this after reading CONTENT-CORE-RULES.md.
This checklist maps strategy to the current live structure.

## Phase 1: Core pages first

1. Home: index the identity and next actions quickly.
- File: index flow is rendered by js/portal-home.js
- Must answer: Who am I, what I do, what I do well, where to click next
- Keep 2 main CTAs: view work, contact

2. About: explain how thinking works.
- Page: /about/
- Must include: domain focus, strengths, process, tools, client fit, selected milestones
- End with practical takeaway, not self-praise

3. Services: explain by problem and outcome.
- Pages:
  - /services/
  - /services/mini-app-design/
  - /services/document-design/
  - /services/creative-media/
- For each page include: what is provided, who it fits, process, deliverables, expected timeline, related cases, CTA

4. Contact: short and clear brief intake.
- Page: /contact/
- Keep form fields short and practical
- Avoid long form and avoid sales pressure tone

## Phase 2: Build trust layer

1. Work section as case studies.
- Pages:
  - /work/
  - /work/mini-apps/
  - /work/document-design/
  - /work/creative-media/
- Each case should include:
  - context
  - problem
  - role
  - solution
  - process
  - result
  - lesson

2. Resources section.
- Page: /resources/
- Build by value type: template, checklist, guide, worksheet, prompt, media pack
- Each resource should state: who it is for, format, usage condition, and expected outcome

3. Insights bridge.
- Page: /insights/
- Use as doorway to blog and future insight taxonomy
- For each article summary: problem, observation, takeaway

## Content quality gate (apply to every page)

- Does the page start from a real problem?
- Does it explain why that problem exists?
- Does it show a different thinking approach?
- Does it explain how the work addresses it?
- Does it end with a takeaway, not a sales push?
- Does it answer at least one hidden question?
- Does it avoid internal labels like focus, capabilities, process, target audience as default UI headings?
- If a heading sounds like a data field name, has it been rewritten into a direct message?

## Hidden question bank

- Why does this exist?
- Why is it designed this way?
- What problem does it solve?
- What did I learn?
- What experiment is next?
