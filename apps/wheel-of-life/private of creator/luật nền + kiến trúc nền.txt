luật nền + kiến trúc nền” phải khóa trước, nếu không càng làm càng loãng, AI càng dễ kéo code đi lệch ngữ cảnh.

Em sẽ dựng cho anh một bộ khung chuẩn bị trước khi build, theo đúng tinh thần: chặt, scale được, audit được, dễ cho AI bám, và không tự mâu thuẫn khi mở rộng.

1. THỨ TỰ ƯU TIÊN PHẢI KHÓA TRƯỚC

Đừng build UI hay tính năng trước. Nên khóa theo thứ tự này:

Tầng 1 — Product Truth
App này là gì
Không phải là gì
User vào để làm gì
Kết quả cuối cùng user nhận được là gì
AI đóng vai gì và không được vượt vai gì
Tầng 2 — Domain Model
Các domain chính của app
Ranh giới mỗi domain
Quan hệ giữa các domain
Cái gì là dữ liệu gốc, cái gì là dữ liệu sinh ra
Tầng 3 — Interaction Logic
User đi từ đâu tới đâu
AI được phép can thiệp đoạn nào
Khi nào cho chọn từ khóa, khi nào cho AI hỏi tiếp
Khi nào ghi dữ liệu, khi nào chỉ tạm suy nghĩ
Tầng 4 — Data Structure
Lưu gì
Lưu ở cấp nào
Quan hệ giữa node, flow, answer, reflection, audit, AI output
Tầng 5 — UI + Scaffold
UI chỉ là lớp trình bày
Component nào là generic
Component nào là domain-specific
Quy tắc layout và state
Tầng 6 — Cursor Rules + AI Rules
AI được sửa gì
Không được phá gì
Khi thêm mới phải theo khuôn nào
Khi audit phải check gì
2. PRODUCT TRUTH — PHẢI VIẾT THÀNH FILE GỐC

Cần có 1 file gốc, ví dụ:

/docs/product/PRODUCT_TRUTH.md

Nội dung phải khóa 7 thứ:

2.1 App này là gì

Ví dụ:

Công cụ coaching/self-reflection dạng guided keyword exploration
Dẫn người dùng từ cảm nhận hiện tại tới nhận diện vấn đề, sau đó map vào wheel of life nâng cấp
2.2 App này không phải là gì
Không phải app chữa trị tâm lý
Không phải app bói
Không phải chatbot nói chuyện lan man
Không phải app AI trả lời thay người dùng toàn bộ
Không phải quiz đơn thuần
2.3 Giá trị cốt lõi
Giúp user tự nhận diện đúng hơn
Giúp user thấy mối liên kết giữa các vấn đề
Giúp user lưu lại tiến trình hiểu bản thân
Giúp AI tiếp tục khai vấn trên đúng ngữ cảnh đã mở
2.4 Đầu ra chính
Một bản đồ tự nhận diện
Một trạng thái wheel of life theo ngữ cảnh thật
Một lịch sử reflection
Một chuỗi node vấn đề đã đi qua
Một bộ dữ liệu cá nhân để tiếp tục phát triển
2.5 Vai trò của AI
AI là người dẫn mở nghĩa
AI giúp diễn giải, nhóm vấn đề, đặt câu hỏi tiếp
AI không được kết luận tuyệt đối về con người
AI không được áp đặt phán xét
2.6 Đơn vị tương tác chính
Keyword node
Reflection prompt
Life area
Context branch
AI assist session
2.7 Nguyên tắc cốt lõi
Start from felt reality
Expand gradually
Preserve context chain
Never flatten nuance
Every conclusion must trace back to chosen context

5. SCAFFOLD RULES — PHẢI CÓ TRƯỚC KHI AI CODE

Anh nên có ít nhất 4 file rules.

5.1 AI_AGENT_ENFORCER.md

File này ở gốc project.

Nó nói cho AI biết:

không được tạo code ngoài cấu trúc
không được trộn domain
không được viết logic business trong UI
không được sửa schema tùy tiện
mọi thay đổi lớn phải cập nhật docs tương ứng
5.2 MANDATORY_APP_SCAFFOLD_RULES.md

File này cũng ở gốc project.

Chứa rule cứng:

domain-first
feature compose from domain
engine separate from UI
all AI prompts centralized
all schemas versioned
every node type must be typed
every mapping must be traceable
5.3 UI_SYSTEM_RULES.md
component naming
spacing scale
icon usage
card behavior
responsive logic
empty states
state visibility rules
5.4 DATA_AND_AI_RULES.md
schema versioning
prompt versioning
AI output cannot overwrite raw user data
derived data must be marked
every AI insight needs source context references
6. CURSOR RULES — NÊN VIẾT THEO NHÓM

Không nên viết 1 file rule dài hỗn hợp. Nên tách.

Nhóm A — Architectural Rules
Domain separation
No cross-domain hidden coupling
No business logic inside presentational components
Engine layer mandatory for flow logic
Nhóm B — Data Rules
All persistent structures must use schema
No ad hoc JSON blobs in random files
Every stored record must include source, timestamp, version
Nhóm C — UI Rules
UI must reveal gradually
Never overload first screen
All steps should feel discoverable
Each card/action must represent a real domain entity
Nhóm D — AI Rules
AI responses must be context-bounded
AI cannot jump to deep advice without enough context
AI must reference selected nodes / user reflections
AI should prefer questions before conclusions
Nhóm E — Audit Rules
Every new branch must be auditable
Every mapping must have reverse lookup
Orphan nodes are forbidden
Contradictory node linkage must be flagged
